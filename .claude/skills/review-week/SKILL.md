# Еженедельный разбор (MASTER_SPEC §25, §10.10)

1. Разреши все прогнозы с вышедшим сроком (`--mode resolve`).
2. По каждому промаху: открой протокол прогона в journal/funnel_logs/, определи по контрфактам Дирижёра, КАКОЙ голос завёл систему не туда. Объяснение обязательно — «не знаю почему» не принимается; «данные были недоступны» принимается с фиксацией пробела.
2b. Дайджест разборов пользователя `/debate`: `python orchestrator/run.py --mode challenge-digest --since <дата прошлого разбора>`. Это мостик «дельные вопросы пользователя → предложения» (§25). Учитываются ТОЛЬКО live-разборы (mock исключён, П16). Повторяющиеся слабые критерии рубрики → кандидаты на усиление промптов/рубрики; повторяющиеся пробелы в данных → кандидаты на новые источники/коннекторы. ДИСЦИПЛИНА §10: единичный разбор — наблюдение, НЕ поправка; предлагать только при повторяемости.
3. Сформулируй ПРЕДЛОЖЕНИЯ поправок в journal/proposed_adjustments.md с датой (включая выводы из дайджеста разборов п.2b). ЗАПРЕЩЕНО менять weights.yaml на этом шаге — применение только через /apply-weights.
4. Обнови журнал ловушек journal/traps.jsonl: были ли пропущенные манипуляции за неделю.
5. Обнови дашборд: Brier по корзинам, hit rate по школам, стоимость инсайта, предотвращённые ошибки.
6. Доложи пользователю сводку недели: исходы, главный урок, накопленные предложения.
