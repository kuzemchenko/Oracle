# PLAN — каскадо-первичная воронка (event → cascade → instrument)

Статус: ПРЕДЛОЖЕН, ждёт утверждения пользователя. Код не трогается до «да».
Дата: 2026-06-15. Автор: Дирижёр. Канон: MASTER_SPEC.md (§5, §6, §9, §14, §17, §23, §24).

## 0. Зачем
Текущая реализация итерирует **закрытый список из 14 тикеров** (`orchestrator/context.py:29` CORE,
`config/universe.yaml`) и якорит каждый скан к Brent. Это отклонение от канона:
- **§6 Эт.1** требует скан по *событиям/трендам/аномалиям* (200–500), не по тикерам.
- **§17.2** требует *события 1–4 порядка → консенсус → запечатанный прогноз*.
- **§14** задаёт универсум **свойствами** (ликвидно/торгуемо/есть данные/есть шорт), не списком.

Поэтому контур ищет «инструмент → новости про него → каскад» (задом наперёд), упирается в
эффективные Brent-ETF и за 10 живых прогонов выдал 0 идей (см. [[funnel-empty-days-diagnosis]]).

## 1. Принцип: брандмауэр «открытие vs запечатывание»
- **Пространство ОТКРЫТИЯ — открыто.** Скан событий и построение каскадов не ограничены никаким
  списком тикеров. Думать можно о любом эффекте в мире.
- **Пространство ЗАПЕЧАТЫВАНИЯ — ограничено реальностью §9/П16.** Запечатать форвард-прогноз можно
  только на инструмент с источником цены, который мы тянем (EODHD). НО этот набор **динамический**
  (любой тикер EODHD резолвится на лету), а не замороженные 14.
- Узел каскада без торгуемого/разрешимого инструмента → **лист ожидания §17 / research-only**
  (паттерн SPCX), не выбрасывается.

## 2. Амплитуда узла — детерминированная (инвариант 6: математика не LLM)
LLM **только предлагает** причинную связь (узел→узел). Силу считает код:
- **shock_0** — квантованный первоисточник (движение корневого фактора в σ, или интенсивность
  news-кластера в z к фону).
- **sensitivity(node)** — историческая эластичность/бета доходностей инструмента узла к shock корня,
  с лагом из `knowledge/causal_links.yaml` (там уже есть эмпир. corr+lag, напр. USO↔BNO 0.9467 lag0).
  Оценка — walk-forward регрессией на ≥10 лет (§23.1, честная зона), с дисперсией оценки.
- **amplitude(node) = sensitivity × shock_0 × transmission_reliability(цепочки)**.
- **probability(node)** — форвардная вер-ть достичь порога: gaussian_baseline_realized_vol (как в
  `orchestrator/calibrate.py`) ⊗ надёжность звена цепочки.
- **П8:** если sensitivity не оценивается (мало истории / связи нет в библиотеке) → «нет данных»,
  узел без торгуемого утверждения → watchlist/research. Никаких выдуманных бет.

## 3b. Ревизия 18.06.2026 — «поток интересных идей» (утв. пользователем)
Уточнение принципа §1 после разбора живого прогона (контур опять схлопнулся к BNO.US):

- **Главный продукт — ранжированный ПОТОК интересных идей**, а не «то, что можно запечатать».
  Всё, что движок счёл интересным по эджу, показывается связной картиной §8. Ничего не прячется.
- **Реестр 4 тем перестаёт быть воротами в анализ.** Первичный путь — каскадный движок по ВСЕМ
  событиям (LLM предлагает цепочку 1–4 порядка; амплитуду/вероятность считает код, инвариант 6).
- **Резолв — в КОНКРЕТНЫЕ компании, не в секторный ETF.** Эдж по §5/П5 — на дальних чокпоинт-узлах
  (CLF, GEV, FCX…), не на широком прокси. EODHD отдаёт отдельные акции. ETF — лишь осознанный выбор
  «широкая экспозиция», не потолок. `proxy_etf`-схлопывание убирается.
- **Ранжирование по РАСЧЁТНОМУ эджу** (тектонический потенциал × амплитуда), не по салиентности
  кластера. Дорогой 21-школьный разбор — только на топ-K по эджу.
- **Некалиброванное ≠ неторгуемое.** Торгуемую некалиброванную компанию полностью анализируем и
  ПОКАЗЫВАЕМ в потоке (research-метка). В анализ пускаем (снять hard-reject `theme_focused`).
- **Запечатывание — тихий служебный ТЕГ поверх потока, не ворота.** Отмечает, какие идеи ещё и в
  форвард-Brier-треке. Идея не запечатана ≠ скрыта. seal — только калиброванное подмножество (П16).
- **Лист ожидания §17.2 — узкий:** только когда торговать физически нечем сейчас (инструмента нет /
  катализатор в будущем, паттерн SPCX-до-IPO). Не свалка для некалиброванного.
- **Граница честности (держим):** не подавать сфабрикованную «калиброванную вероятность» без фона
  (П8); не засчитывать некалиброванные идеи в форвард-Brier-трек (П16). Картина — полная, с меткой.

## 3c. Ревизия 18.06.2026 — отрезание первого порядка + покозвенный прогноз (утв. пользователем «да»)
Заострение §3b после живого прогона `funnel_20260618T073001Z`: все 6 кандидатов = BNO/USO/DBC/COPX —
первый порядок и сплошь `proxy_etf`/`related` реестра. Подтверждено: контур итерирует список тем.

- **D1. 14 тикеров — снять роль «кандидат/мишень», оставить «бенчмарк/фон».** Список НЕ удаляется
  (он — бенчмарк §30 + калиброванные фоны + данные, что мы тянем). Убирается его роль потолка
  открытия и дна резолва: `proxy_etf`-схлопывание выпиливается, резолв идёт в КОНКРЕТНУЮ компанию
  через динамический EODHD. ETF допустим, лишь когда терминальный узел реально и есть сектор.
- **D2. Первый порядок на базовом инструменте — вон из потока; seal только на калибруемом терминале.**
  Уже-в-цене монетки (тип BNO-шорт после −3%) не «ниже рангом», а отрезаются от потока идей. seal
  не цепляется к первому порядку. Осознанный размен: путь к воротам Б→Д (270 исходов) замедляется —
  продукт важнее скорости ворот (развязка §3b держится).
- **D3. Прогноз неочевидной идеи = условный сценарий по звеньям, не одна калиброванная p.** Каскадный
  путь раскладывается на звенья; каждое — своим честным ярусом, с провенансом; свёртка детерминирована
  (инвариант 6). Ярусы честности звена (надёжность, лаг, gain):
    - **A — эмпирический:** история есть (`causal_links.yaml` corr+lag / walk-forward бета). Как есть.
    - **B — структурный:** ценовой корреляции нет, но есть измеримая экспозиция (доля выручки от
      рынка, доля затрат, узкое место мощности). gain = экспозиция × операционный рычаг (с провенансом
      и источником — П8). Здесь живут неочевидные дальние компании.
    - **C — только механизм:** LLM утверждает связь, данных нет → низкий приор надёжности + широкая
      полоса, метка «механизм, не подтверждён». Это звено атакует слепой суд. Если весь edge на
      C-звене → идея research-only (без калиброванной p), но В ПОТОКЕ показывается.
  Свёртка (код): `amplitude = shock₀ × Π gainᵢ`, `reliability = Π reliabilityᵢ`, `lag = Σ lagᵢ`,
  **дисперсия оценок пробрасывается** (длинная цепь с одним слабым звеном → широкая полоса → честное
  «низкая уверенность», не уверенное враньё). **edge = амплитуда, ещё НЕ отыгранная на терминале**
  (расчётное движение − уже реализованное); ранжируем по непрокинутому edge × надёжность, не по
  салиентности кластера. Дорогой 21-школьный разбор — только на топ-K по edge.
  **Два выхода на идею (П16):** (1) ДОСЬЕ ЭДЖА — всегда, на каждую идею (путь, провенанс звеньев,
  амплитуда+полоса, тайминг, «кто продаёт», что неизвестно); (2) КАЛИБРОВАННАЯ p + seal — только когда
  у терминала есть честный фон; иначе сценарный payoff с вероятностями звеньев, research-метка, не в
  Brier-треке. Никакой выдуманной калиброванной p без фона (П8).

Реализация: расширить `mathlib/cascade.py` (ярус A уже есть: бета+амплитуда+gaussian-p+§9-гейт) —
добавить ярусы B/C, многозвенную свёртку с пробросом дисперсии, shock₀ и edge-vs-отыгранное. Затем
по гейтам §3 (Этапы 1, 3–7). Математика раньше промптов (порядок сборки CLAUDE.md).
