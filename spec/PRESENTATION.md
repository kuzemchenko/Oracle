# Слой подачи отчёта §8 в Telegram

*Документ описывает, КАК отчёт показывается человеку. Это слой ОТОБРАЖЕНИЯ — он не меняет
ни данные, ни инварианты CLAUDE.md. При конфликте формулировок канон — `spec/MASTER_SPEC.md`.*

## Что это и чего НЕ трогает

Цель: боевой отчёт §8 в Telegram должен читаться человеком без знания спецификации.
Меняется ТОЛЬКО текст, который видит пользователь.

**Не меняется (инварианты):**
- Состав 13 полей §8 — это ДАННЫЕ, остаются как есть.
- `journal/predictions.jsonl` и `journal/funnel_logs/` пишутся ПОЛНОСТЬЮ (хук `guard_journal`
  не обходится). Слой подачи только ЧИТАЕТ `funnel_logs`, ничего в журналы не пишет.
- П8, запечатывание, веса, скоринг, денежные ворота, пауза 24 ч §12 — не затрагиваются.

## Где живёт

- `ops/bot_reports.py` — рендер (`format_report`, `format_mock_check`, `format_weak_day`,
  `format_trigger_alert`, бюджет). Здесь же словари `field_help[1..13]`, `_FIELD_TITLES`,
  `_ASSET` (тикер → человеческое имя/описание).
- `config/presentation.yaml` — настройки подачи (правятся командой `/format` или руками).
- `ops/bot.py` — отправка (`_tick_reports`, `_send_card`), команда `/format`.

## Канонический вид (режим long)

```
📈 Идея дня: <имя> (<тикер>) — ставка на рост/снижение
Насколько уверены: ~XX% (против ~50% «вслепую»).      ← вероятность судьи (поле 3), ОДНА цифра

📋 Полный разбор (13 полей §8):

N. <короткий заголовок поля>
   ℹ️ <что это поле и зачем — из field_help[N]>        ← показывается по режиму (см. ниже)
   → <содержание простыми словами; пусто → «[поле пустое]» (П8)>
...
⚖️ «Принять» откроется через 24 ч (пауза §12).
· тех.id <run_id>
```

Правила шапки:
- **Одна** цифра уверенности = вероятность судьи (поле 3). Если её нет — «пока не оценена».
- Сводный балл `0.58/1.00` в шапке НЕ показывается НИКОГДА. Он только в поле 9, с пометкой
  «это НЕ вероятность из п.3, а сводная оценка по 6 критериям».
- Пустое содержание поля → честный маркер `[поле пустое]`, ничего не выдумываем (П8).

## Режимы (config/presentation.yaml)

| Ключ | Значение |
|---|---|
| `mode` | `long` (по умолчанию) — ℹ️ у ВСЕХ полей; `short` — ℹ️ только у полей из `still_unclear`. |
| `still_unclear` | список номеров полей 1..13, у которых в `short` всё ещё показывать ℹ️. |
| `send_mock_to_telegram` | слать ли проверочные (mock) прогоны. По умолчанию `false`. |

Содержание (`→`) показывается ВСЕГДА в обоих режимах. Скрывается/показывается только ℹ️.

## Команда `/format` (без правки кода руками)

- `/format` — показать текущий режим и помеченные поля.
- `/format long` | `/format short` — переключить режим.
- `/format unclear 3 5` — пометить поля 3 и 5 как «ещё мутно» (ℹ️ для них в short остаются).
- `/format clear` — снять все пометки; `/format clear 3` — снять с поля 3.

## Проверочные (mock/test) прогоны

mock НЕ маскируется под идею. По умолчанию (`send_mock_to_telegram: false`) — вовсе не шлётся.
Если включить — приходит с шапкой:
```
🔧 Проверка связи — это НЕ идея.
Тестируем, что трубопровод работает и формат собирается. Содержательные поля пустые.
```

## Тексты ℹ️ (field_help)

Все формулировки ℹ️ — в одном словаре `field_help[1..13]` в `ops/bot_reports.py`. Правка
формулировки = одна строка там же; на данные не влияет.
