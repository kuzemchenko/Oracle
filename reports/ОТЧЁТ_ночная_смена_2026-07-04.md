## 2. Replay-режим — долг F3/П2а закрыт полностью

`sudo -u oracle .venv/bin/python orchestrator/run.py --mode replay --cutoff 2026-06-20`
- Детерминированный граф каскадов на дату: шок корня, realized/vol терминалов, гейт баров И
  чувствительности звеньев — всё по барам ≤ cutoff (ноль look-ahead, П16).
- Тест-гарантия: расчёт с asof на полной БД (с подложенным «диким будущим») бит-в-бит равен
  расчёту на физически усечённой БД.
- LLM не вызывается, ничего не запечатывается, протоколы в отдельном journal/replay_logs/
  (бот и дашборд их не видят). Это сырьё для будущего П2б-тюнинга без затрат на токены.

## 3. Известные баги — все четыре закрыты

1. **cascade-лаг** — направленная семантика (directed-записи приоритетны по всему списку).
2. **portfolio месячный риск** — 0.0 больше не выдаётся за известный накопленный риск: охват
   «только текущий прогон» заявлен явно; параметр под реальные суммы готов к этапу Д.
3. **cost=None** — LLM-вызовы без цены видимы («бюджет_прогона.без_стоимости» в протоколе) и
   ПЛАТНЫ для стопа-на-лету (оценка по среднему известных; раньше 100 таких вызовов при
   потолке $1 не остановили бы прогон — нашёл ночной кросс-контроль).
4. **Тихое падение cron** — обёртка ops/cron_alert.sh: ненулевой код боевого прогона →
   journal/alerts.jsonl → бот пушит «🚨 прогон упал» с ретраем доставки.

## 5. MEDIUM-пакет дневного ревью + бюджет §24 + challenge-хардненинг

- Протоколы прогонов пишутся атомарно (обрыв не оставляет битый JSON); sqlite timeout=30 в 12
  местах («database is locked» при пересечении cron/бота — ждём, не падаем).
- Бюджетный стоп на лету журналируется честным «ОСТАНОВ», а не «тема вне универсума».
- «не лонг» больше не превращается в запечатанный прогноз стороной ВВЕРХ (матч по токенам,
  отрицание = отказ формализовать); синонимы направлений в карте корреляций.
- **/debate (live) обходил бюджетный контур ПОЛНОСТЬЮ** — теперь пред-проверка + стоп-на-лету
  + новый суб-потолок challenge $2/прогон (существующие потолки не тронуты).
- **Внутренние воронки event_first не платили во внешний потолок** (реальный лимит был
  k×cap+cap) — цепочка гардов + ре-эскалация стопа.
- challenge.py прошёл 16 кругов адверсарного кросс-ревью: точный матч тикера (раньше «SO»
  находил USO.US), no-clobber протоколов через O_EXCL, единый канонический ключ времени
  (UTC strftime, календарная валидация — фикстуры больше не «новее» боевых), терпимость к
  битым формам, аудируемые отказы. Два предложения ревьюера отклонены с письменным
  обоснованием (см. протокол арбитража).
