# FIX 2026-07-15 — «ключ ещё не заведён»: механика поля «внимание» (запрос владельца)

## Проблема (владелец, 15.07)
«Внимание толпы: не измерено» — постоянный тезис выдачи. Датчик перегретости (П2а) на
практике мёртв: в дайджесте 15.07 измерено ~2 идеи из 10+.

## Диагноз (по коду, 3 дыры)
1. **Идеи треков (money/provisional/digest_only) не получают кандидатов ключа вообще** —
   `attention_field.annotate_ideas` даёт им только сиды (11 активов) и реестр. Любой новый
   актив узла графа → «ключ Trends не назначен (нет сида/реестра/кандидатов)» навсегда.
   При этом у ТЕМЫ цепочки, породившей узел, ключи уже есть (`config/news.yaml
   trends_keywords`) и данные по ним УЖЕ фетчатся ежедневно в 06:45.
2. **Ключ, назначенный в прогоне (из слов кластера картографа), фетчится только завтра** —
   `data/trends.load_keywords` включает реестр в суточный фетч 06:45, а прогон в 09:00.
   К завтрашнему фетчу событие протухло, у новых событий новые ключи → тезис
   «появятся после суточного фетча» воспроизводится вечно.
3. **Качество ключей кластера**: кандидат = первый сырой токен кластера. В реестре
   зафиксировано (пересдача запрещена, §R0#5): VST.US→«vampires», ZIM.US→«strikes»,
   EBS.US→«eastern». Ключ бессмыслен для Trends → поле меряет мусор.

## Фикс (эта ветка, fix/attention-keys)
- **Т1. Кандидаты для треков от темы цепочки**: `attention_field.track_candidates_for(треки,
  universe)` — узел трека → `_chain` (боевая форма: ВНУТРИ `s["node"]`, ревью 15.07 БЛОКЕР;
  верхний уровень — фолбэк) → тема универсума (`themes.<t>.cascade_chain`) → ПЕРВЫЙ
  `trends_keywords` темы из news.yaml; для цепочек КАРТОГРАФА (вне тем) — ключи их
  новостного кластера (chain_keys, тема приоритетнее). Ключи тем уже в канонике → поле
  меряется в ТОТ ЖЕ день. Узел вне тем/карт — честное «не_измерено». `digest_only`
  (= отсев graph_select) цепочку НЕ несёт вовсе → кандидатов у отсева нет (сужение спеки,
  решение ревью 15.07: отсев не запечатывается, датчик там наименее ценен). Сиды/реестр
  приоритетнее кандидата (порядок resolve_key не меняется).
- **Т2. Дофетч новых ключей в прогоне**: `annotate_ideas(..., fetcher=)` — после первого
  прохода ключи со статусом «нет канонических строк…/данные появятся после суточного
  фетча» фетчатся сразу (`data/trends.fetch_keywords_now`, канон-окно, кэп 6, пауза как у
  суточного фетча), поля пересчитываются. Ошибки фетча — fail-soft (поле остаётся
  «не_измерено», прогон живёт). mock/fix_keys=False — фетч не вызывается (П16).
- Покрытие §R5 считается ПОСЛЕ дофетча.

## НЕ в этой ветке (нужна подпись владельца — правка подписанного правила §R0#5)
- **Т4. Пересдача мусорных ключей**: правило «первый ключ окончателен» заперло
  «vampires»/«strikes»/«eastern». Предложение: разовая журналируемая пересдача (старая
  запись остаётся в провенансе, причина обязательна, только вручную с подписью).

## Критерии готовности
- Тесты: кандидаты треков от темы; сид приоритетнее кандидата; дофетч пересчитывает поле
  в том же вызове; кэп; fail-soft; mock не фетчит; билдер track_candidates_for
  (+ цепочки картографа через ключи их кластеров, тема приоритетнее).
- Живой прогон: у идей треков тематических цепочек поле «внимание» = ok в день прогона.
